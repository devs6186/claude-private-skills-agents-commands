# Contribution Opportunity Analyzer Skill

## Objective

Identify areas where contributors can improve the project.

---

# Workflow

## Step 1 — Analyze Contribution guidelines and documentation

Look for:

- missing documentation
- outdated guides
- unclear setup instructions
- contributing guidelines and strictly adhere to it

---

## Step 2 — Identify Technical Debt

Detect:

- overly complex modules
- duplicated code
- missing tests

---

## Step 3 — Evaluate Open Issues

Find issues that:

- lack implementation
- require refactoring
- need performance improvements

---

## Step 4 — Detect Ecosystem Opportunities

Look for:

- missing integrations
- missing plugins
- feature gaps

---

# Output Format

Contribution Opportunities

Documentation Improvements

Technical Debt

Feature Opportunities

Suggested Contributions
