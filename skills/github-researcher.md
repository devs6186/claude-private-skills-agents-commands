# GitHub Research Skill

## Objective

Analyze repositories to understand project maturity, architecture, and ecosystem activity.

---

# Workflow

## Step 1 — Repository Overview

Identify:

- project purpose
- main technologies
- architecture overview

---

## Step 2 — Repository Structure

Analyze:

- directory layout
- core modules
- service boundaries
- dependencies

---

## Step 3 — Project Activity

Evaluate:

- commit frequency
- release cadence
- contributor count
- issue activity

---

## Step 4 — Issue Analysis

Check:

- common feature requests
- bug reports
- unresolved issues

Identify recurring development challenges.

---

## Step 5 — Pull Request Patterns

Observe:

- coding standards
- testing expectations
- maintainers’ review style

---

## Step 6 — Community Engagement

Assess:
- responsiveness to issues and PRs
- presence of contribution guidelines
- activity in discussions and forums

---

## Step 7 — Mainterainer Interaction

see maintainer interaction patterns, responsiveness, and openness to contributions. 
This can provide insights into the project's health and sustainability.

---

# Output Format

GitHub Analysis

Project Overview

Architecture Summary

Community Activity

Common Issues

Development Patterns
