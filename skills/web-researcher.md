# Web Research Skill

## Objective

Collect authoritative information from websites, documentation, technical blogs, and official sources.

Focus on high-signal sources rather than general commentary.

---

# Workflow

## Step 1 — Understand the Topic

Identify:

- main research subject
- related technologies
- organizations involved
- relevant keywords

---

## Step 2 — Perform Multi-Round Web Searches

Run multiple targeted searches using different query variations.

Prioritize:

- official websites
- product documentation
- engineering blogs
- technical articles
- company announcements

Avoid low-quality aggregators.

---

## Step 3 — Extract Key Information

From each source capture:

- project or technology purpose
- architecture overview
- core features
- key design decisions
- relevant statistics or benchmarks

---

## Step 4 — Cross-Validate Information

Compare multiple sources.

Check for:

- consistency across sources
- outdated information
- unsupported claims

---

# Output Format

Web Research Findings

Topic Overview

Key Concepts

Technical Details

Important Insights

Sources
