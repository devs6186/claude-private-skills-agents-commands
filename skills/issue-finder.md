# Issue Finder Skill

## Objective

Identify relevant issues in a repository and categorize them.

---

# Workflow

## Step 1 — Scan Open Issues

Collect issues from the repository.

Focus on:

- open issues
- recently active issues
- high-comment threads

---

## Step 2 — Categorize Issues

Classify issues into categories:

- bugs
- feature requests
- performance issues
- documentation improvements
- enhancement proposals
- tough and high quality issues

---

## Step 3 — Identify Beginner-Friendly Issues

Look for tags such as:

- good-first-issue
- help-wanted
- beginner-friendly

---

## Step 4 — Identify High-Impact Issues

Find issues that:

- affect core functionality
- have many comments
- involve architectural changes

---
