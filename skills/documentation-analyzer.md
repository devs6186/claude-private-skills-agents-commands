# Documentation Analyzer Skill

## Objective

Extract authoritative technical knowledge directly from official documentation.

Documentation is the most reliable source for understanding:

- architecture
- APIs
- configuration
- limitations
- recommended usage patterns

---

# Workflow

## Step 1 — Locate Official Documentation

Find:

- official docs website
- API reference pages
- architecture guides
- getting started guides
- advanced usage documentation

Prioritize official documentation over third-party tutorials.

---

## Step 2 — Extract Core Concepts

Identify:

- main architectural components
- system design principles
- key features
- supported integrations

---

## Step 3 — Identify Implementation Patterns

Look for:

- recommended project structure
- configuration patterns
- common usage examples
- deployment models

---

## Step 4 — Identify Limitations

Extract documented:

- limitations
- warnings
- performance considerations
- unsupported features

---

Documentation Analysis

Technology Overview

Core Architecture

Key Features

Limitations

Important Notes
