# Repo Structure Analyzer Skill

## Objective

Understand how a repository is structured and identify its architectural layout.

---

# Workflow

## Step 1 — Scan Directory Structure

Identify:

- top-level directories
- service folders
- modules
- packages

---

## Step 2 — Identify Core Components

Determine:

- main application code
- supporting libraries
- configuration files
- infrastructure code

---

## Step 3 — Detect Entry Points

Look for files such as:

- main application files
- server startup scripts
- CLI entry points

---

## Step 4 — Identify Architecture Pattern

Determine if the project uses:

- layered architecture
- modular architecture
- microservices
- plugin architecture

---

Repository Structure
