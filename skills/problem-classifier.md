# Problem Classifier Skill

## Objective

Determine the type of technical issue described by the user.

---

# Workflow

## Step 1 — Analyze Problem Description

Identify signals such as:

- error messages
- logs
- stack traces
- configuration files
- environment details

---

## Step 2 — Classify Problem Type

Possible categories:

- runtime error
- dependency conflict
- configuration issue
- API integration problem
- performance issue
- test failure
- environment issue
