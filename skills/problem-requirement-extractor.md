# Problem Requirement Extractor

## Objective

Extract clear functional and non-functional requirements from the user's problem description.

---

# Workflow

1. Identify core problem
2. Determine system goals
3. Identify functional requirements
4. Identify non-functional requirements

Examples:

- scalability
- latency
- security
- reliability

---

# Output Format

Requirements Analysis

Problem Summary

Functional Requirements

Non Functional Requirements
